📝 Developer Prompt: Connect Existing Checkout Pages to Stripe

We already have:

Three checkout pages on arnetrice.com (Starter Hosting, Growth Hosting, Enterprise Hosting).

Each page lists the plan details and optional add-ons.

All products, add-ons, and setup fees have been created in Stripe, with price IDs available in .env.test, .env.prod, and .env.

🎯 What’s Missing

The checkout pages do not talk to Stripe yet.
We need to wire them into Stripe Checkout so that when a customer selects a plan and add-ons, everything is sent to Stripe in one unified checkout session.

✅ Implementation Steps

Frontend: Send Selections to Backend

When the customer clicks Complete Purchase:

Collect the base plan (starter/growth/enterprise).

Collect the billing cycle (monthly/annual).

Collect any selected add-ons.

POST this data as JSON to the backend (e.g., /create-checkout-session).

Backend: Create Checkout Session

Parse the incoming payload.

Look up the correct Stripe price IDs from environment variables or config.

Always include the base plan.

Append selected add-ons.

If an add-on has a setup fee, append its one-time product.

Frontend: Redirect to Stripe

When the backend returns {"url": session.url}, redirect the customer to Stripe Checkout.

Webhook Handling

Add /webhook to capture Stripe events:

checkout.session.completed → mark subscription active.

invoice.paid → confirm subscription billing.

Verify with STRIPE_WEBHOOK_SECRET from env.

✅ Deliverables for Developers

Customers start from the existing checkout pages.

Their selections (plan, billing cycle, add-ons) are POSTed to backend.

Backend builds a single Stripe Checkout session with all line items (plan + add-ons + setup fees).

Customer is redirected to Stripe Checkout to pay securely.

Webhook listens for success events to update our system.
